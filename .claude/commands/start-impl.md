---
description: 新機能の実装セッションを開始
argument-hint: [feature-name]
---

新機能「$1」の実装セッションを開始します。

以下の手順で進めてください:

1. **セッション用ディレクトリの作成**
   - 最新の連番を確認し、`tmp/{連番}_{機能名}/` を作成
   - 例: `tmp/001_$1/`

2. **計画ファイルの作成**
   - `tmp/XXX_$1/plan.md` を作成
   - 以下の内容を含めてください:
     - 機能の概要
     - 実装手順（ステップバイステップ）
     - 影響を受けるファイル
     - テスト計画

3. **履歴ファイルの作成**
   - `tmp/XXX_$1/prompt.md` を作成
   - 初期状態として、このセッションの開始時刻と目的を記録

4. **実装計画の提示**
   - 作成した `plan.md` の内容をユーザーに提示
   - 実装を開始する前に、必ずユーザーの許可を得てください

**重要な注意事項**:
- コードの変更は、`plan.md` が承認されるまで実施しないでください
- 作業完了ごとに、テスト・型チェック・フォーマットを必ず実施してください
- `prompt.md` は各作業ステップで更新してください
