# cr-create スキル

## 目的
- 実装が終わったブランチから GitHub の Change Request（Pull Request）を作成し、関係者へ変更内容を共有する。
- 進行中タスクのレビューを円滑にするため、要約・テスト結果・関連 Issue を丁寧に記述する。

## 実行フロー
1. **準備**: `git status` で作業ツリーがクリーンであることを確認し、最新のテスト結果を把握する。
2. **タイトル整備**: 変更種類（fix/feat/chore など）と簡潔な説明を組み合わせたタイトルを作成する。
3. **本文テンプレ**: 下記セクションを埋める。
   - `## 概要`: 箇条書きで変更点を説明（なぜ必要か／どう解決したか）。
   - `## テスト`: 実行したコマンドを列挙し、必要に応じて失敗理由も記載。
   - 末尾に `Closes #<issue number>` を入れて Issue を自動クローズ。
4. **PR 作成**: `gh pr create --title ... --body ...` で送信。`origin` 以外のリモートを使う場合は `--head` を調整。

## 言語ポリシー
- **PR タイトルと本文は必ず日本語で記述すること。**
- 例外なく概要・テスト欄も日本語で書き、必要なら英語の補足を括弧で添える。
- 自動生成系コマンドのテンプレを使う場合も、送信前に日本語化を忘れない。

## ベストプラクティス
- テストが失敗した場合は、その時点の開発環境制約やエラーメッセージを `※` などで簡潔に説明。
- 画像・動画など大きな添付が必要なときは GitHub コメントや Issue へリンクし、PR 本文はテキスト中心に保つ。
- 固定表現（例: `Closes #42`）を入れ忘れないようにテンプレの雛形を用意しておく。
