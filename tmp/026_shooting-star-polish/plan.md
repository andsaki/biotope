# 実装計画: shooting-star-polish

## 目的
直近で追加・調整された流れ星エフェクトを、実機確認しながら「見えるが主張しすぎない」状態に仕上げる。
初回表示タイミング、夜空での視認性、モバイルでの負荷、既存の星・照明・季節エフェクトとのなじみを確認し、必要な範囲だけ微調整する。

## 実装スコープ
### 1. 作成するファイル
- なし

### 2. 変更するファイル
- `src/constants/shootingStars.ts`
  - 流星の周期、初回遅延、表示時間、光跡サイズ、透明度、軌道を必要に応じて調整する。
- `src/components/ShootingStars.tsx`
  - 実機確認で問題が出た場合のみ、描画ロジック、テクスチャ、opacity 制御を最小限修正する。
- `src/components/Stars.tsx` / `src/components/ReflectedStars.tsx`
  - 流れ星との重なりや明るさの競合が明確な場合のみ調整対象にする。
- `docs/ai-guide/feature.md` または関連 docs
  - 挙動や運用上残すべき知見が出た場合のみ追記する。

## 実装手順（ステップバイステップ）
1. 現在の流れ星実装と定数を確認し、変更候補を整理する。
2. ローカル dev server を起動し、夜間表示で流れ星の初回出現、周期、光跡の長さ、明るさを確認する。
3. デスクトップとモバイル相当 viewport で、流れ星が画面内に入り、UI や水辺表現を邪魔しないか確認する。
4. 必要な場合のみ `shootingStars.ts` の定数を調整する。
5. 定数調整だけで解決しない問題がある場合のみ `ShootingStars.tsx` を修正する。
6. `npm run build` と可能な範囲の表示確認を実施する。
7. `prompt.md` に実施内容、確認結果、残課題を記録する。

## 技術的詳細
- 既存の React Three Fiber / Three.js 実装を維持する。
- 流星本体は `PlaneGeometry` + `CanvasTexture` + additive blending の現行方式を前提にする。
- まず定数調整で解決し、ロジック変更は必要な場合に限定する。
- 新しい依存関係は追加しない。
- パフォーマンス影響を抑えるため、追加 geometry や per-frame allocation は増やさない方針にする。
