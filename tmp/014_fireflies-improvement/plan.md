# 実装計画: fireflies-improvement

## 目的
- 水面上空を漂うホタルの演出を「飛ぶ ↔ 着地」を繰り返すライフサイクルに変更し、睡蓮や植木鉢にも止まって光るようにして情緒を高める。
- 点滅や移動スピードをチューニングし、ブルームを活かした幻想的な光り方に整える。
- instanced mesh の更新コストを抑えつつ、描画パラメータを一定 FPS に依存しないよう安定させる。

## 実装スコープ
### 1. 作成するファイル
- なし（既存 Fireflies コンポーネントの改修のみ）

### 2. 変更するファイル
- `src/components/Fireflies.tsx`
  - 水草データの取り込み・休憩スポット一覧の定義
  - ホタル状態（resting/flying）の追加と初期化処理
  - `useFrame` 内での状態遷移・水面/植物の揺れ連動・点滅処理
  - instanced mesh の色/行列バッファ更新・マテリアル設定

## 実装手順（ステップバイステップ）
1. 水面・睡蓮に関する定数を import し、RESTING_SPOTS と新たなタイミング定数（rest/fly duration、IDEAL_FRAME_TIME 等）を定義する。Firefly 型に休憩関連フィールドを追加し、初期化で一部を睡蓮/植木鉢へ配置できるようにする。
2. `useFrame` のロジックを delta ベースに書き換え、resting と flying の分岐、睡蓮の波・植木の揺れを反映した位置更新、スポット間の遷移を実装する。基準位置のドリフトとクリッピングも調整する。
3. 点滅フェーズの進行と `brightness` の算出を刷新し、instancedMatrix/instanceColor の Float32Array を再利用して更新する。マテリアルで `toneMapped: false` を指定しブルームを正しく効かせる。

## 技術的詳細
- `LILY_DATA` から睡蓮座標を取り出し、植木鉢も含む RESTING_SPOTS を生成。type/index を付与して後段で波計算に利用する。
- `IDEAL_FRAME_TIME` を基準とした `deltaFactor` を導入し、60fps 未満でも飛行速度が破綻しないようにする。
- 睡蓮上の resting 時は `WATER_HEIGHT_BASE`＋波動パラメータを使って上下動。植木鉢上では簡易的な回転揺れを加える。
- 点滅は `sin` の正値のみを二乗して強く発光させ、負値域では 0.05 へ抑えて「消灯」状態を再現する。
- `instancedBufferAttribute` の色バッファを `Float32Array` で保持し、毎フレーム `needsUpdate` を立てることで個別の発光強度を適用する。
